Project Summary

The objective of this project was to attempt to see what the general Twitter user base thinks of the United States 2016 Presidential Election and how candidates portray themselves on the platform.

In order to simplify the scope of analysis and to mediate with the (messy) nature of twitter data, I limited the scope of tweets to those that could be explicitly found on candidate profiles. I further simplified general user tweets to those that explicitly mention candidates in their tweets via hashtags (ex. #HillaryClinton or #BernieSanders). Lastly, in order to further simplify the analysis, I steered away from attempting to analyze user sentiment towards candidates and instead focused on the sentiment of candidate accounts.

In order to add dimensionality to candidate sentiment I also included an additional layer of analysis, mainly visualizations based on the terms and hashtags most commonly associated with each candidate.

Though distinct from the initial sentiment analysis foundation, this additional layer of analysis helps shape and also complicate the manner in which candidates portray themselves and how they are perceived.

Technologies

  • Python + NLTK
  • MatPlotLib + PyPlot + Numpy
  • Tweepy (Twitter API)

Methods Overview

Created a sentiment analysis application using a Naive Bayes Classifier that analyzes Candidate Twitter data inorder to assign a Negative/Positive Score to each candidate. All twitter data was obtained using Tweepy API Streaming and Search.

Also created simple text visualizations to provide a more comprehensive view of general public sentiment toward each candidate using #hashtag term frequencies.
